## ⚠️ Active Issues

### 1. Energy Dissipation Theory Mismatch (HIGH PRIORITY)

**Status:** 🔴 BLOCKING physics validation

**Problem:**
Numerical energy dissipation rate does not match theoretical prediction:
- **Theory:** dH/dt = -η ∫ J² dV
- **Observed:** dH/dt ≈ -0.03 × η ∫ J² dV  (96-99% error)
- **Factor:** ~30x discrepancy

**Evidence:**
```
Test: η = 1e-4, 200 steps
Step   0: dH/dt = -1.46e+00 (theory: -7.24e+07)  [100% error]
Step  40: dH/dt = -2.19e-03 (theory: -6.50e-02)  [97% error]
Step 160: dH/dt = -1.88e-03 (theory: -6.50e-02)  [97% error]
```

**Possible Causes:**
1. **Hamiltonian definition:**
   - Code uses: `H = compute_hamiltonian(psi, omega, grid)`
   - Should use: `H = compute_hamiltonian(psi, phi, grid)` where φ from ∇²φ = ω
   
2. **Current density definition:**
   - `compute_current_density` returns J_φ (toroidal current density)
   - Theory formula may use different J (e.g., ∂H/∂ψ)
   
3. **Volume integral:**
   - Current: `dV = R * dr * dθ`
   - Missing factors? Check Jacobian

4. **Theory derivation:**
   - Formula dH/dt = -η ∫ J² dV may be incorrect
   - Need to re-derive from first principles

**Impact:**
- ✅ Stability NOT affected (energy still decreases monotonically)
- ❌ Cannot claim "physics correct"
- ⚠️ Blocks full validation

**Next Steps:**
1. Theoretical review session to re-derive energy dissipation
2. Check Hamiltonian definition (psi, omega) vs (psi, phi)
3. Verify current density normalization
4. Compare with published MHD codes

**Assigned:** 小P ⚛️  
**Priority:** HIGH (blocks v1.3 completion)  
**Est. Time:** 2-3 hours (theory + verification)

### 2. IMEX Performance (LOW PRIORITY)

**Status:** 🟡 Known limitation

**Problem:**
IMEX solver is slow:
- 100 steps: ~3 minutes
- 800 steps: ~30 minutes (estimated)

**Cause:**
- Each time step: 2× implicit solve (GMRES on 64×64 grid)
- Typical convergence: 5-10 iterations per solve
- No preconditioning

**Impact:**
- Long validation tests (e.g., convergence test)
- Not suitable for production RL training

**Mitigation:**
- Use explicit solver for small η (η < 1e-5)
- Reduce grid size for validation tests
- Future: implement preconditioner or multigrid

**Assigned:** Backlog  
**Priority:** LOW (optimization, not correctness)

## ✅ Resolved Issues

### ~~Resistive Instability~~ (FIXED 2026-03-19)

**Problem:** Resistive term caused instability (NaN) for η ≥ 1e-5

**Root Cause:** Wrong sign in implicit solver:
```python
# WRONG (before fix):
psi_new = psi_star + dt/2 * eta * J(psi_new)  # Anti-damping!

# CORRECT (after fix):
psi_new = psi_star - dt/2 * eta * J(psi_new)  # Damping ✅
```

**Fix:**
- Changed sign in `implicit_resistive.py`
- Updated operator: `(I + dt*eta*[J op])` → `(I - dt*eta*[J op])`
- Verified: stable for η up to 1e-3 (100 steps)

**Validation:** Test 1 (Stability Range) PASS

**Status:** ✅ RESOLVED
