## v1.3-revised: Complete 2D Reduced MHD ⭐ NEW

### Goal

**Primary:** Implement complete reduced MHD with advection-diffusion coupling

**Why change from scalar z±:**
- Scalar z±=ω±ψ 不是真Elsässer (wrong physics)
- Transfer to v2.0 only 1.5/6 (ineffective stepping stone)
- **Better: 学正确的[ψ,φ] MHD → transfers to (z∓·∇)z± in v2.0**

### Physics

**Complete Reduced MHD:**
```python
∂ψ/∂t = [ψ,φ]_tor + η∇²ψ
∂ω/∂t = [ω,φ]_tor + (1/R²)[ψ,J] + ν∇²ω

where:
  φ = ∇⁻²ω  (stream function, single Poisson solve)
  J = -∇²ψ  (current density)
  [f,g]_tor = (1/R)(∂f/∂r·∂g/∂θ - ∂f/∂θ·∂g/∂r)  (Poisson bracket)
```

**New additions vs v1.2:**
1. ✅ Poisson bracket [ψ,φ] (advection)
2. ✅ Coupling term (1/R²)[ψ,J] (force balance)
3. ✅ Single stream function φ (correct physics)

**Physics correctness:**
- Force balance: J×B = ∇P maintained
- PyTokEq Solovev equilibrium should hold
- Energy conservation improved

### Integrator: IMEX Advection-Diffusion

**Explicit (RK2 or AB2):**
```python
rhs_explicit = [ψ,φ]_tor  # Nonlinear advection
             + (1/R²)[ψ,J]_tor  # Coupling
```

**Implicit (Crank-Nicolson):**
```python
(I - η·dt/2·∇²)ψ^{n+1} = ψ^n + dt·rhs_explicit
(I - ν·dt/2·∇²)ω^{n+1} = ω^n + dt·rhs_explicit
```

**Stability:** Unconditionally stable (IMEX)

### Implementation Plan

**Phase 1: Operators (3 days)**
- Add `poisson_bracket_toroidal(f, g, grid)` (~2h)
  - Implementation: Arakawa scheme (energy-conserving)
  - Test: Analytical bracket properties
- Verify `poisson_solver` works for φ=∇⁻²ω
- Unit tests: bracket + solver

**Phase 2: Solver Core (1 week)**
- Implement complete RHS (advection + coupling)
- IMEX integrator (reuse `integrators/symplectic.py` structure)
- Boundary conditions (same as v1.2)
- Integration tests: stability, energy

**Phase 3: PyTokEq Validation (3 days)**
- Generate Solovev IC (ψ_eq, ω_eq)
- Run equilibrium evolution (should be static)
- Verify force balance holds
- Compare energy drift vs v1.2

**Phase 4: RL Integration (5 days)**
- Update environment (same obs/action as v1.2)
- Train PPO with advection physics
- Compare performance vs v1.2 baseline
- Verify RL learns real MHD dynamics

**Phase 5: Documentation (3 days)**
- Completion report
- Physics validation
- v1.2 vs v1.3-revised comparison

**Total: ~1.5 months** (conservative, 可能更快如code reuse充分)

### Success Criteria

## v2.0: Vector Elsässer Production

### Goal

**True 3D vector Elsässer MHD-RL**

### Physics

**Vector Elsässer:**
```python
z⁺ = u + B  (3D vector fields)
z⁻ = u - B

∂z⁺/∂t = -(z⁻·∇)z⁺ - ∇P + ν∇²z⁺
∂z⁻/∂t = -(z⁺·∇)z⁻ - ∇P + ν∇²z⁻

∇·u = 0  (incompressibility)
∇·B = 0  (divergence-free)
```

**Cross-helicity:**
```python
H_C = ∫ u·B dV = ∫ (|z⁺|² - |z⁻|²)/4 dV
```

### Key Additions vs v1.4

**A1: Scalar → Vector**
- (ψ,ω) scalars → (u,B) vectors
- 2 fields → 6 components (u_r,u_θ,u_ζ, B_r,B_θ,B_ζ)

**A2: Advection upgrade**
- [ψ,φ] Poisson bracket → (z∓·∇)z± vector advection
- Structure learned in v1.3 ✅
- Implementation: 3D gradient + dot product

**A3: Incompressibility**
- ∇·u = 0 constraint
- Pressure P from projection method
- Poisson solve for P

**A4: Complete physics**
- J×B Lorentz force
- Curvature terms (toroidal)
- Realistic dissipation

### Implementation

**Phase 1: Vector Fields (1 month)**
- u, B vector storage
- Vector operators (grad, div, curl)
- (z∓·∇)z± advection

**Phase 2: Incompressibility (3 weeks)**
- Projection method
- Pressure Poisson solver
- ∇·u=0 enforcement

**Phase 3: RL Integration (3 weeks)**
- 12-18D action (3D z± forcing)
- 30-40D observation
- Multi-obj training

**Phase 4: Validation (2 weeks)**
- H_C conservation
- Benchmark vs M3DC1
- Production testing

**Total: 3 months**

### Deliverables

1. Vector Elsässer solver validated
2. H_C conservation < 5%
3. RL multi-obj trained
4. Publication-ready results

## Risk Assessment

### v1.3-revised Risks

**R1: Poisson bracket complexity**
- Arakawa scheme需careful implementation
- Mitigation: 参考cylindrical code (已exists)
- **Risk: LOW** (已有reference)

**R2: PyTokEq equilibrium unstable**
- Coupling term可能引入instability
- Mitigation: Start with weak coupling, ramp up
- **Risk: MODERATE**

**R3: Timeline optimistic**
- 1.5月可能→2月 (if debug needed)
- Mitigation: Conservative estimate已adopted
- **Risk: LOW**

### v1.4 Risks

**R4: 3D geometry complexity**
- ζ dimension adds computation + complexity
- Mitigation: Start 2.5D (low ζ resolution)
- **Risk: MODERATE**

**R5: 6D RL convergence**
- High-dim action space challenge
- Mitigation: Curriculum learning (2D→4D→6D)
- **Risk: MODERATE-HIGH**

### v2.0 Risks

**R6: Scalar→Vector transition**
- Big conceptual jump
- Mitigation: v1.3 [ψ,φ] structure teaches foundation
- **Risk: MODERATE**

**R7: Incompressibility numerical**
- Projection method stability
- Mitigation: 参考literature (standard method)
- **Risk: LOW-MODERATE**

### Overall Risk: LOW-MODERATE ✅

**Why acceptable:**
- 每个milestone ≤2 new things
- Incremental validation
- Fallback options available
