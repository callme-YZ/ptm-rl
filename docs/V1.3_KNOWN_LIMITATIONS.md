## 1. Energy Dissipation Quantitative Error (20-45%)

### Description

Numerical energy dissipation rate differs from theoretical prediction by 20-45%, depending on initial conditions.

### Theory

For resistive MHD:
```
dH/dt = -η ∫ J² dV
```

where:
- `η` = resistivity
- `J = Δ*ψ/R` = toroidal current density (Grad-Shafranov operator)
- `dV = r·R dr dθ dφ` = toroidal volume element

### Observed Behavior

**Smooth initial conditions:**
```python
psi = r²(1-r/a)² + small perturbation
→ Error ≈ 20%
```

**Sharp initial conditions:**
```python
psi = r²·cos(θ)
→ Error ≈ 45%
```

### Example Results

**Test case:** η = 1e-4, dt = 1e-4, grid 32×64

**Smooth IC:**
```
Step 10-40:
  dH/dt_numeric ≈ -1.1e-4
  dH/dt_theory  ≈ -1.4e-4
  Error = 21%
```

**Sharp IC:**
```
Step 2-5:
  dH/dt_numeric ≈ -2.5e+1
  dH/dt_theory  ≈ -4.6e+1
  Error = 45%
```

### Root Cause Analysis

**Implicit backward Euler scheme:**
```
(I - dt·η·[J operator])·ψ^(n+1) = ψ*
```

Solved with GMRES, which linearizes the nonlinear operator `J(ψ) = Δ*ψ/R` around current iterate.

**Verification test (linear vs nonlinear):**

| Operator | Error |
|----------|-------|
| Linear Laplacian (`∂ψ/∂t = η·Δψ`) | 6% ✅ |
| Nonlinear GS (`∂ψ/∂t = -η·J`, `J=Δ*ψ/R`) | 45% ❌ |

**Conclusion:** Error comes from implicit treatment of **nonlinear** Grad-Shafranov operator, not from volume element or theory formula.

### Impact

**Physics validation:**
- ✅ Direction correct: energy decreases
- ✅ Order of magnitude correct: O(-10) to O(-100)
- ⚠️ Quantitative mismatch: factor 1.25-2×

**RL training:**
- ✅ Reward signal direction correct
- ✅ Relative comparisons valid (comparing different policies)
- ⚠️ Absolute reward values biased

**Diagnostics:**
- ✅ Can detect energy decrease trend
- ❌ Cannot precisely predict dissipation rate

### Mitigation Strategies

**1. Use smooth initial conditions (recommended)**
```python
# Good: smooth profile
psi = r²(1-r/a)² + perturbation
→ Error ≈ 20%

# Avoid: sharp gradients
psi = r²·cos(θ)
→ Error ≈ 45%
```

**2. Accept as systematic bias**
- Document in reports
- Compare relative changes (policy A vs policy B)
- Don't rely on absolute dissipation values

**3. Use explicit scheme (not recommended)**
- Pro: matches theory better (~5% error)
- Con: dt limited to ~1e-6 (vs 1e-4 with IMEX)
- Con: 100× slower

**4. Hybrid approach (future work)**
- Use simpler Laplacian approximation: `J ≈ Δψ`
- Pro: 6% error
- Con: less physical (missing 1/R factor and GS structure)

### Status

**Accepted as known limitation for v1.3**

Reason:
- Physics direction correct
- Stable evolution
- Sufficient for RL reward signal
- Fixing would require major algorithmic changes (Newton iteration, multigrid, etc.)

## 2. Computational Cost

### Description

IMEX solver is ~5× slower than pure symplectic integration.

### Benchmark

**Grid:** 32×64  
**Hardware:** Apple M1

| Solver | Time/step | Relative |
|--------|-----------|----------|
| Symplectic only (ideal MHD) | 0.10 sec | 1× |
| **IMEX (resistive MHD)** | **0.50 sec** | **5×** |

### Breakdown

**IMEX step components:**
1. Poisson solve (`∇²φ = ω`): ~0.20 sec (40%)
2. GMRES (implicit resistive): ~0.15 sec (30%)
3. Gradient computations: ~0.10 sec (20%)
4. Other (BC, integration): ~0.05 sec (10%)

### Bottlenecks

**GMRES iterations:**
- Each iteration: compute `J(ψ) = Δ*ψ/R`
- Grad-Shafranov operator expensive (finite differences + division by R)
- Typically 5-20 iterations per solve

**Poisson solver:**
- Sparse direct solve: O(N^1.5) for 2D
- Building Laplacian matrix once (cached)
- Repeated solves for different RHS

### Impact

**RL training:**
- 10k episodes × 100 steps/episode = 1M steps
- 1M steps × 0.5 sec = 139 hours ≈ 6 days

**Acceptable for:**
- Physics validation (< 1000 steps)
- Small-scale RL experiments

**Not acceptable for:**
- Large-scale hyperparameter search
- Real-time control

### Mitigation

**1. Grid resolution trade-off**
```python
# Validation: high resolution
grid = ToroidalGrid(nr=64, ntheta=128)  # slow

# RL training: lower resolution
grid = ToroidalGrid(nr=32, ntheta=64)   # 4× faster
```

**2. Efficient initial guess**
```python
# Good: use previous timestep
x0 = psi_old.ravel()
→ GMRES converges in 5-10 iterations

# Bad: zero initial guess
x0 = zeros
→ GMRES needs 20-50 iterations
```

**3. Future optimizations**
- Multigrid preconditioner for GMRES
- JAX JIT compilation
- GPU acceleration
- Adaptive timestep (larger dt when smooth)

### Status

**Accepted for v1.3, optimize in v1.4**

## 3. Grid Resolution Requirements

### Description

Current default grid (32×64) is adequate for validation but marginal for fine-scale structures.

### Minimum Requirements

**Physics validation:**
- Minimum: 32×32
- Recommended: 32×64 ✅ (current)
- High fidelity: 64×128

**RL training:**
- Minimum: 32×64 ✅ (current)
- Recommended: 64×128
- Production: 128×256 (if computational budget allows)

### Resolution Effects

| Feature | 32×64 | 64×128 | 128×256 |
|---------|-------|--------|---------|
| Energy conservation | ✅ 0.15% | ✅ 0.08% | ✅ 0.04% |
| Poisson bracket accuracy | ⚠️ 3e-3 | ✅ 8e-4 | ✅ 2e-4 |
| Island width | ⚠️ ±10% | ✅ ±3% | ✅ ±1% |
| Computational cost | 1× | 4× | 16× |

### Trade-offs

**32×64 (current choice):**
- ✅ Fast enough for RL training
- ✅ Sufficient physics fidelity
- ⚠️ May miss fine structures (thin current sheets)

**64×128:**
- ✅ Better physics accuracy
- ⚠️ 4× slower (24 days for 10k episodes)

**128×256:**
- ✅ Production quality
- ❌ 16× slower (96 days for 10k episodes)

### Recommendation

**v1.3:** Use 32×64 for RL development  
**v1.4:** Consider adaptive mesh refinement or 64×128 for production

## 4. Boundary Condition Constraints

### Description

Current implementation assumes **conducting wall** boundary condition: `ψ(r=a) = 0`

### Limitation

Cannot model:
- Free boundary equilibria
- Plasma-wall interaction
- Edge localized modes (ELMs)

### Impact

**OK for:**
- Core MHD (tearing modes, sawteeth)
- Ideal MHD stability
- Basic control scenarios

**Not OK for:**
- Vertical displacement events (VDEs)
- Edge physics (pedestal, ELMs)
- Divertor tokamaks

### Mitigation

**Future work (v1.4+):**
- Implement free boundary solver
- Add vacuum region outside plasma
- Green's function for external coils

## 5. Axisymmetry Assumption

### Description

Code assumes axisymmetry: `∂/∂φ = 0` (no toroidal variation)

### Limitation

Cannot model:
- 3D MHD modes (ballooning, kink)
- Toroidal field ripple
- Resonant magnetic perturbations (RMP)

### Impact

**OK for:**
- Reduced MHD (2D poloidal + toroidal average)
- Tearing modes (m ≠ 0, n = 0)
- Vertical stability

**Not OK for:**
- Toroidal modes (n ≠ 0)
- 3D equilibria
- Stellarators

### Mitigation

**Not planned for v1.x**

Reason: 3D MHD requires major code restructuring and 100-1000× computational cost

## 6. Single Fluid MHD

### Description

Resistive MHD treats plasma as single magnetofluid.

### Missing Physics

- Two-fluid effects (electron-ion separation)
- Kinetic effects (Landau damping, resonances)
- Turbulence (micro-instabilities)
- Neoclassical transport

### Impact

**OK for:**
- Macroscopic MHD stability
- Global energy balance
- Basic control demonstrations

**Not OK for:**
- Micro-turbulence (ITG, ETG)
- Bootstrap current
- Banana orbits

### Mitigation

**Out of scope for reduced MHD**

Use gyrokinetic codes (GENE, GS2) for turbulence  
Use full MHD + kinetic closures for hybrid models
