## 1. Observation Space Evolution

### 1.1 v1.1 Observation (9D)

```python
obs_v1_1 = np.concatenate([
    psi_modes,      # (8,) Fourier coefficients of ψ
    [energy_drift], # (1,) dE/dt normalized
])
```

**What it captures:**
- Magnetic field structure (via ψ modes)
- Energy trend (drift)

**What it misses:**
- ❌ Vorticity dynamics (no ω)
- ❌ div(B) constraint (Maxwell violation)
- ❌ Absolute energy (only drift)

**Consequence:** Agent cannot detect full MHD state.

### 1.2 v1.2 Observation (19D)

```python
obs_v1_2 = np.concatenate([
    psi_modes,      # (8,) Poloidal flux Fourier modes
    omega_modes,    # (8,) Vorticity Fourier modes (NEW)
    [energy],       # (1,) Total energy (NEW)
    [energy_drift], # (1,) dE/dt
    [div_B_max],    # (1,) Max div(B) violation (NEW)
])
```

**New components:**

**1. Vorticity modes (8D):**
```python
omega_k = fft2(omega, axes=(1,2))  # 2D FFT
omega_modes = [
    omega_k[0,0],   # m=0, n=0 (mean)
    omega_k[1,0],   # m=1, n=0 (tearing)
    omega_k[2,0],   # m=2, n=0 (kink)
    omega_k[0,1],   # m=0, n=1 (toroidal)
    ...
]
```
**Why:** Vorticity drives dynamics (∂ψ/∂t ~ [ψ,φ] ~ ω).

**2. Total energy (1D):**
```python
E = ∫ (|∇ψ|² + ω²/2) dV
```
**Why:** Absolute level matters, not just drift.

**3. div(B) max (1D):**
```python
div_B = divergence_toroidal(B, grid)
div_B_max = np.max(np.abs(div_B))
```
**Why:** Constraint violation indicator.

### 1.3 Observation Normalization

**Critical for RL training:**

```python
class ObservationNormalizer:
    def __init__(self):
        # Running statistics
        self.mean = np.zeros(19)
        self.std = np.ones(19)
        self.count = 0
    
    def normalize(self, obs):
        """Z-score normalization."""
        obs_norm = (obs - self.mean) / (self.std + 1e-8)
        return np.clip(obs_norm, -10, 10)  # Prevent outliers
    
    def update(self, obs):
        """Online mean/std update."""
        self.count += 1
        delta = obs - self.mean
        self.mean += delta / self.count
        self.std = np.sqrt((self.std**2 * (self.count-1) + delta**2) / self.count)
```

**Why normalize:**
- Different components have different scales (energy ~ 10³, modes ~ 10⁻²)
- Neural networks require similar input scales
- Improves RL convergence

## 2. Multi-Objective Reward Design

### 2.1 The Problem

**Conflicting objectives:**

1. **Performance:** Minimize island width (suppress tearing)
2. **Efficiency:** Minimize power consumption
3. **Safety:** Maintain physics constraints (div(B) < ε)

**Cannot optimize all simultaneously!**

Example:
- Max power → Best performance, worst efficiency
- Zero power → Best efficiency, worst performance

**Need: Pareto-optimal solutions**

### 2.2 Reward Components

**Component 1: Island Width Penalty**

```python
def compute_island_penalty(psi, grid):
    """Detect and measure island width."""
    # Find X-point (saddle point in ψ)
    psi_xx, psi_yy, psi_xy = compute_hessian(psi)
    saddle_points = find_saddles(psi_xx, psi_yy, psi_xy)
    
    if len(saddle_points) > 0:
        # Island present
        w = estimate_island_width(psi, saddle_points[0])
        penalty = -w  # Negative (want to minimize)
    else:
        # No island
        penalty = 0.0
    
    return penalty
```

**Typical values:**
- No island: penalty = 0
- Small island (w ~ 0.01): penalty = -0.01
- Large island (w ~ 0.1): penalty = -0.1

**Component 2: Power Consumption Penalty**

```python
def compute_power_penalty(J_ext, grid, P_max):
    """Penalize high power usage."""
    P_ext = np.sum(J_ext**2) * grid.dV
    
    # Normalized penalty
    penalty = -(P_ext / P_max)  # ∈ [-1, 0]
    
    return penalty
```

**Component 3: Constraint Violation Penalty**

```python
def compute_constraint_penalty(div_B_max, threshold=1e-6):
    """Hard penalty for div(B) violation."""
    if div_B_max > threshold:
        # Exponential penalty
        penalty = -100 * np.exp(div_B_max / threshold)
    else:
        penalty = 0.0
    
    return penalty
```

**Why exponential:**
- Small violations (< 1e-6): tolerable
- Large violations (> 1e-4): catastrophic
- Exponential discourages violations

### 2.3 Multi-Objective Reward Function

**Weighted sum approach:**

```python
def compute_reward(state, action, J_ext):
    """
    Multi-objective reward with tunable weights.
    
    Parameters
    ----------
    state : dict
        Contains psi, omega, div_B_max
    action : np.ndarray
        RL action (current drive amplitudes)
    J_ext : np.ndarray
        External current density
    
    Returns
    -------
    reward : float
        Scalar reward signal
    """
    # Component 1: Island suppression
    r_island = compute_island_penalty(state['psi'], grid)
    
    # Component 2: Power efficiency
    r_power = compute_power_penalty(J_ext, grid, P_max=1.0)
    
    # Component 3: Constraint satisfaction
    r_constraint = compute_constraint_penalty(state['div_B_max'])
    
    # Component 4: Action smoothness (optional)
    r_smooth = -0.01 * np.sum(action**2)  # L2 penalty
    
    # Weighted sum
    w_island = 1.0      # Performance weight
    w_power = 0.1       # Efficiency weight
    w_constraint = 10.0 # Safety weight (high!)
    w_smooth = 0.01     # Regularization
    
    reward = (w_island * r_island + 
              w_power * r_power + 
              w_constraint * r_constraint +
              w_smooth * r_smooth)
    
    return reward
```

**Weight tuning guidelines:**

| Weight | Range | Purpose |
|--------|-------|---------|
| w_island | 1.0-10.0 | Primary objective (performance) |
| w_power | 0.01-1.0 | Efficiency (lower = more aggressive) |
| w_constraint | 10.0-100.0 | Safety (must be high) |
| w_smooth | 0.001-0.1 | Regularization (prevent jitter) |


### 2.4 Pareto Front Exploration

**Challenge:** Different weight combinations → different optimal policies.

**Strategy 1: Sequential Training**

```python
# Train 3 policies with different emphasis
policies = []

# Policy 1: Performance-focused (suppress tearing at any cost)
weights_perf = {'island': 10.0, 'power': 0.01, 'constraint': 10.0}
policy_perf = train_ppo(env, weights=weights_perf, steps=50k)
policies.append(('performance', policy_perf))

# Policy 2: Balanced
weights_bal = {'island': 1.0, 'power': 0.5, 'constraint': 10.0}
policy_bal = train_ppo(env, weights=weights_bal, steps=50k)
policies.append(('balanced', policy_bal))

# Policy 3: Efficiency-focused (minimize power)
weights_eff = {'island': 1.0, 'power': 2.0, 'constraint': 10.0}
policy_eff = train_ppo(env, weights=weights_eff, steps=50k)
policies.append(('efficiency', policy_eff))

# Deployment: Choose based on scenario
if power_budget_low:
    use_policy = policy_eff
elif tearing_critical:
    use_policy = policy_perf
else:
    use_policy = policy_bal
```

**Strategy 2: Multi-Task RL (Advanced)**

Train single policy conditioned on desired trade-off:

```python
# Augment observation with weight vector
obs_augmented = np.concatenate([obs, weight_vector])

# Policy learns: "If w_power high → use less current"
policy = train_conditioned_ppo(env, obs_augmented, steps=100k)

# Deployment: Specify desired trade-off
weight_vector = [1.0, 0.8, 10.0]  # Island, power, constraint
action = policy(obs, weight_vector)
```

**Strategy 3: Reward Curriculum**

Start with simple objective, gradually add complexity:

```python
# Week 1-2: Learn basic control (island only)
weights_phase1 = {'island': 1.0, 'power': 0.0, 'constraint': 10.0}

# Week 3-4: Add power penalty
weights_phase2 = {'island': 1.0, 'power': 0.1, 'constraint': 10.0}

# Week 5-6: Full multi-objective
weights_phase3 = {'island': 1.0, 'power': 0.5, 'constraint': 10.0}
```

**Why curriculum helps:**
- Simpler problem first → faster initial learning
- Gradual complexity → stable training
- Avoids reward hacking (e.g., zero action for zero power)

## 3. Reward Shaping Techniques

### 3.1 Dense vs Sparse Rewards

**Sparse (v1.1 style):**
```python
# Only reward at episode end
if episode_done:
    reward = -final_island_width
else:
    reward = 0.0
```
**Problem:** Hard for RL to learn (credit assignment)

**Dense (v1.2 recommended):**
```python
# Reward at every step
reward = -current_island_width - 0.1 * power
```
**Benefit:** Immediate feedback, faster learning

### 3.2 Potential-Based Shaping

**Idea:** Add progress indicator without changing optimal policy.

```python
def potential(state):
    """State value estimate (e.g., energy to ideal equilibrium)."""
    psi_target = grad_shafranov_solution(q_profile)
    distance = np.linalg.norm(state['psi'] - psi_target)
    return -distance  # Higher potential = closer to target

def shaped_reward(state, action, next_state):
    # Original reward
    r = compute_reward(state, action)
    
    # Add potential difference (Ng et al., 1999)
    gamma = 0.99  # Discount factor
    F = gamma * potential(next_state) - potential(state)
    
    return r + F
```

**Theorem (Ng et al.):** Optimal policy unchanged by potential-based shaping.

### 3.3 Intrinsic Motivation (Exploration Bonus)

**For sparse tearing events:**

```python
def curiosity_bonus(obs, obs_next, action):
    """Reward visiting novel states."""
    # Forward model prediction error
    obs_next_pred = forward_model(obs, action)
    error = np.linalg.norm(obs_next - obs_next_pred)
    
    # Bonus for surprise (model uncertainty)
    bonus = 0.01 * error
    
    return bonus

reward_total = reward_task + curiosity_bonus
```

**Why useful:** Encourages exploring different current drive patterns.

### 4.2 Reward Scaling

**Problem:** Different reward components have different magnitudes.

**Solution:** Target total reward ~ O(1)

```python
# Before scaling
r_island = -0.05    # Island width ~ 0.05
r_power = -0.3      # Power ~ 0.3
r_constraint = -0.0 # No violation

# After scaling

# → Total ~ -0.65 (O(1) range)
```

**Heuristic:** Tune weights so typical |reward| ∈ [0.1, 10].

### 5.2 Evaluation Metrics

**Metric 1: Island Width Suppression**
```python
w_0 = initial_island_width
w_final = final_island_width_after_control

suppression_ratio = (w_0 - w_final) / w_0
# Higher = better (1.0 = complete suppression)
```

**Metric 2: Power Efficiency**
```python

# Higher = better (more suppression per watt)
```

**Metric 3: Constraint Satisfaction**
```python

# Lower = better (0.0 = perfect)
```

**Success criteria:**
- RL policy beats all baselines in at least 2 out of 3 metrics
- OR achieves Pareto-optimal trade-off (e.g., 80% suppression with 50% less power than baseline)

## 6. Implementation Plan

### 6.1 Observation Extraction

```python
class ObservationExtractor:
    def __init__(self, grid, n_modes=8):
        self.grid = grid
        self.n_modes = n_modes
    
    def extract(self, solver):
        """Extract 19D observation from solver state."""
        # Fourier modes
        psi_k = fft2(solver.psi)
        omega_k = fft2(solver.omega)
        
        psi_modes = self._select_modes(psi_k, n=8)
        omega_modes = self._select_modes(omega_k, n=8)
        
        # Global quantities
        energy = solver.compute_energy()
        energy_drift = solver.compute_energy_drift()
        
        # Constraint
        div_B = divergence_toroidal(solver.B, self.grid)
        div_B_max = np.max(np.abs(div_B))
        
        # Concatenate
        obs = np.concatenate([
            psi_modes.real,
            omega_modes.real,
            [energy],
            [energy_drift],
            [div_B_max],
        ])
        
        return obs  # Shape: (19,)
```

### 6.2 Reward Computation

```python
class RewardComputer:
    def __init__(self, weights):
        self.w_island = weights['island']
        self.w_power = weights['power']
        self.w_constraint = weights['constraint']
    
    def compute(self, state, action, J_ext):
        """Compute multi-objective reward."""
        r_island = self._island_penalty(state['psi'])
        r_power = self._power_penalty(J_ext)
        r_constraint = self._constraint_penalty(state['div_B_max'])
        
        reward = (self.w_island * r_island +
                  self.w_power * r_power +
                  self.w_constraint * r_constraint)
        
        # Debug info
        info = {
            'reward_island': r_island,
            'reward_power': r_power,
            'reward_constraint': r_constraint,
            'reward_total': reward,
        }
        
        return reward, info
```

### 6.3 Gym Environment Integration

```python
class ToroidalMHDEnv(gym.Env):
    def __init__(self, reward_weights=None):
        super().__init__()
        
        # Defaults
        if reward_weights is None:
            reward_weights = {'island': 1.0, 'power': 0.5, 'constraint': 10.0}
        
        # Components
        self.obs_extractor = ObservationExtractor(grid, n_modes=8)
        self.reward_computer = RewardComputer(reward_weights)
        self.obs_normalizer = RunningMeanStd(shape=(19,))
        
        # Spaces
        self.observation_space = gym.spaces.Box(
            low=-np.inf, high=np.inf, shape=(19,), dtype=np.float32
        )
        self.action_space = gym.spaces.Box(
            low=-1.0, high=1.0, shape=(6,), dtype=np.float32
        )
    
    def step(self, action):
        # MHD evolution
        J_ext = self.current_drive.compute_current(action)
        self.solver.step(action)
        
        # Extract observation
        obs = self.obs_extractor.extract(self.solver)
        obs_norm = self.obs_normalizer.normalize(obs)
        
        # Compute reward
        state = {
            'psi': self.solver.psi,
            'omega': self.solver.omega,
            'div_B_max': obs[18],  # Last component
        }
        reward, info = self.reward_computer.compute(state, action, J_ext)
        
        # Termination
        done = (self.step_count >= self.max_steps) or self._is_unstable()
        
        # Update normalizer
        self.obs_normalizer.update(obs)
        
        return obs_norm, reward, done, info
```

## 11. Risk Mitigation

### 11.1 If Multi-Objective Training Unstable

**Backup:** Single-objective first, then multi-objective fine-tuning
```python
# Phase 1: Island suppression only (10k steps)
weights_phase1 = {'island': 1.0, 'power': 0.0, 'constraint': 10.0}

# Phase 2: Add power penalty (fine-tune 5k steps)
weights_phase2 = {'island': 1.0, 'power': 0.5, 'constraint': 10.0}
```

### 11.2 If Reward Hacking Occurs

**Example hack:** Agent learns "do nothing" → zero power, zero island change → high efficiency

**Fix:** Add progress requirement
```python
# Must reduce island below threshold within N steps
if island_width > 0.05 and step > 500:
    reward -= 10.0  # Failure penalty
```

### 11.3 If Pareto Front Degenerate

**Problem:** All policies converge to same solution

**Fix:** Increase weight diversity
```python
# Use logarithmic spacing
w_power_values = [0.01, 0.1, 1.0, 10.0]  # Wide range
```
